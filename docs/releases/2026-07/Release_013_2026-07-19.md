## 🎯 Tổng quan mục tiêu
Bản phát hành này tập trung vào việc hoàn thiện hệ thống quản lý phương tiện truyền thông thông qua việc tích hợp tính năng tải lên ảnh, trích xuất dữ liệu EXIF và quản lý liên kết rút gọn. Đồng thời, hệ thống cũng được tối ưu hóa về mặt kiến trúc dữ liệu và cấu hình môi trường để đảm bảo tính ổn định và hiệu năng cao hơn trong quá trình phát triển.

## ✨ Chi tiết thay đổi

### 🚀 Tính năng mới
- <kbd>a29680c</kbd> **Quản lý Shortlink**: Bổ sung các API hỗ trợ liệt kê (dạng slice), truy vấn chi tiết, xóa và xem trước liên kết rút gọn.
- <kbd>a595400</kbd> **Trích xuất Metadata**: Tích hợp `MediaMetadataExtractor` để tự động thu thập dữ liệu EXIF và lưu trữ vào thực thể Media khi tải ảnh lên.
- <kbd>e16e648</kbd> **Tải lên và lưu trữ ảnh**: Triển khai chức năng tải lên file ảnh kèm theo cơ chế lưu trữ file và lưu thông tin vào cơ sở dữ liệu.

### 🛠 Bảo trì hệ thống & Refactor
- <kbd>45a206c</kbd> **Sửa lỗi Swagger**: Cấu hình lại schema cho tính năng tải lên multipart bằng cách sử dụng `@RequestPart` và chú thích OpenAPI chuẩn xác.
- <kbd>7bd0489</kbd> **Tối ưu định danh**: Chuyển đổi kiểu dữ liệu của định danh (ID) từ UUID sang Long trong thực thể Album để tối ưu hóa hiệu năng.
- <kbd>75967a0</kbd> **Tự động hóa Timestamp**: Cập nhật các lớp thực thể (Entity) sử dụng `@CreationTimestamp` để tự động ghi nhận thời gian tạo bản ghi.
- <kbd>f3b5869</kbd> **Hoàn thiện cấu hình**: Hoàn tất cấu hình biến môi trường `.env`, dọn dẹp thực thể `AlbumMedia` và thêm phụ thuộc H2 cho môi trường kiểm thử.
- <kbd>21e56e7</kbd> **Cấu hình môi trường**: Thiết lập file `.env`, bổ sung `DotenvConfig` và cập nhật datasource cho quá trình kiểm thử.
- <kbd>17a45b3</kbd> **Chuẩn hóa phiên bản**: Cố định phiên bản Java 17 và tối giản hóa cấu trúc của thực thể `AlbumMedia`.
- <kbd>846bdc0</kbd>, <kbd>a9aacd5</kbd>, <kbd>1adb716</kbd>, <kbd>b190d74</kbd> **Định dạng code**: Tự động định dạng mã nguồn bằng Spotless để đảm bảo tuân thủ tiêu chuẩn code chung.
