## 🎯 Tổng quan mục tiêu
Bản phát hành này tập trung vào việc hoàn thiện hệ thống quản lý dữ liệu đa phương tiện với tính năng tải ảnh mới và trích xuất siêu dữ liệu EXIF tự động. Đồng thời, chúng tôi đã tối ưu hóa cấu trúc mã nguồn thông qua việc chuẩn hóa xử lý ngoại lệ và tái cấu trúc các dịch vụ lưu trữ để nâng cao tính ổn định và khả năng bảo trì của hệ thống.

## ✨ Chi tiết thay đổi

### 🚀 Tính năng mới
- <kbd>a29680c</kbd> **Mở rộng API Shortlink**: Bổ sung các endpoint cho phép liệt kê (slice), xem chi tiết, xóa và xem trước liên kết rút gọn.
- <kbd>a595400</kbd> **Trích xuất siêu dữ liệu**: Tích hợp `MediaMetadataExtractor` để tự động thu thập thông tin EXIF và lưu trữ vào thực thể Media khi tải lên.
- <kbd>e16e648</kbd> **Hỗ trợ tải ảnh**: Triển khai chức năng tải lên tệp tin kết hợp lưu trữ file vật lý và lưu vết thông tin vào thực thể Media.

### 🛠 Bảo trì hệ thống
- <kbd>d829601</kbd> **Tái cấu trúc I/O**: Chuyển đổi logic đọc/ghi file preview sang lớp dịch vụ thông qua `MediaPreview` record giúp tách biệt tầng logic.
- <kbd>b7c35a2</kbd> **Chuẩn hóa xử lý ngoại lệ**: Chuyển đổi cơ chế xử lý lỗi sang `GlobalExceptionHandler` giúp Controller trở nên tinh gọn và đồng bộ.
- <kbd>7bd0489</kbd> **Thay đổi kiểu dữ liệu ID**: Cập nhật định danh (ID) của thực thể Album từ UUID sang kiểu Long để tối ưu hóa hiệu năng truy vấn.
- <kbd>75967a0</kbd> **Tự động hóa Timestamp**: Cập nhật các lớp thực thể sử dụng `@CreationTimestamp` giúp tự động hóa việc lưu vết thời gian khởi tạo.
- <kbd>f3b5869</kbd> **Cấu hình & Kiểm thử**: Hoàn thiện cấu hình `.env`, dọn dẹp các liên kết AlbumMedia và bổ sung H2 database để phục vụ kiểm thử đơn vị.
- <kbd>846bdc0</kbd>, <kbd>a9aacd5</kbd>, <kbd>1adb716</kbd> **Định dạng code**: Tự động định dạng mã nguồn bằng Spotless nhằm đảm bảo quy chuẩn code style thống nhất.

### 🐛 Sửa lỗi
- <kbd>45a206c</kbd> **Sửa Schema Swagger**: Điều chỉnh cấu trúc multipart upload sử dụng `@RequestPart` và annotation OpenAPI để tài liệu API hiển thị chính xác.
