## 🎯 Tổng quan mục tiêu
Bản phát hành này tập trung vào việc hiện đại hóa cấu trúc dự án sang mô hình monorepo, giúp tích hợp ứng dụng Flutter Frontend một cách đồng bộ. Đồng thời, chúng tôi đã tối ưu hóa quy trình xử lý media, cải thiện độ tin cậy của tài liệu API và thiết lập nền tảng CI/CD vững chắc để tăng tốc độ phát triển và triển khai hệ thống.

## ✨ Chi tiết thay đổi

### 🚀 Tính năng mới
- <kbd>80baa46</kbd> **Tích hợp Frontend**: Triển khai ứng dụng Flutter vào cấu trúc monorepo để hợp nhất quản lý mã nguồn.
- <kbd>7d8b207</kbd> **Dịch vụ Thumbnail**: Thêm `ThumbnailService` tự động tạo ảnh thu nhỏ kích thước 400x400 định dạng JPEG khi tải lên dữ liệu.

### 🐛 Sửa lỗi
- <kbd>1f219d3</kbd> **Cấu hình HTTPS**: Cấu hình lại URL server cho Swagger và thiết lập hỗ trợ trust proxy để đảm bảo tính bảo mật khi chạy qua HTTPS.
- <kbd>daca16a</kbd> **Sửa lỗi Swagger**: Khôi phục chức năng chọn tệp tin trên Swagger bằng cách sử dụng `@Parameter` với định dạng binary thay vì `@RequestBody`.
- <kbd>45a206c</kbd> **Cập nhật Schema Upload**: Chỉnh sửa cấu trúc multipart upload trên Swagger bằng cách sử dụng `@RequestPart` và các chú thích OpenAPI tương ứng.

### 🛠 Bảo trì hệ thống
- <kbd>6aa2c03</kbd> **Tái cấu trúc Monorepo**: Tổ chức lại thư mục dự án thành cấu trúc monorepo với hai thư mục riêng biệt cho backend và frontend.
- <kbd>15e10e3</kbd> **Thiết lập CI/CD**: Tích hợp quy trình CI/CD tự động hóa và bổ sung Dockerfile để hỗ trợ triển khai hệ thống.
- <kbd>eb038d4</kbd> **Cập nhật MediaPreview**: Chuyển đổi logic truy xuất dữ liệu xem trước sang kiểu dữ liệu `MediaPreview` giúp đồng bộ mã nguồn.
- <kbd>d829601</kbd> **Tối ưu I/O File**: Di chuyển logic xử lý đọc/ghi file xem trước vào service thông qua `MediaPreview` record để tách biệt nghiệp vụ.
- <kbd>b7c35a2</kbd> **Xử lý ngoại lệ**: Chuyển logic xử lý ngoại lệ vào `GlobalExceptionHandler` giúp mã nguồn tại các controller trở nên tinh gọn và dễ bảo trì hơn.
