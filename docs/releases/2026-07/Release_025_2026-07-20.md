## 🎯 Tổng quan mục tiêu
Bản cập nhật này tập trung vào việc hiện đại hóa cấu trúc dự án sang mô hình Monorepo, giúp tối ưu hóa quy trình quản lý mã nguồn giữa Frontend và Backend. Đồng thời, chúng tôi tích hợp các tính năng xử lý ảnh nâng cao và cải thiện hạ tầng CI/CD để tăng cường hiệu suất vận hành và trải nghiệm phát triển phần mềm.

## ✨ Chi tiết thay đổi

### 🚀 Tính năng mới
- <kbd>1ce3e90</kbd> **Giao diện thư viện ảnh**: Triển khai màn hình hiển thị thư viện ảnh chính trên nền tảng web, tích hợp hoàn chỉnh với API backend.
- <kbd>80baa46</kbd> **Khởi tạo Flutter Frontend**: Chính thức bổ sung dự án Flutter vào cấu trúc Monorepo để đa dạng hóa nền tảng hỗ trợ.
- <kbd>7d8b207</kbd> **ThumbnailService**: Bổ sung dịch vụ tự động tạo ảnh thu nhỏ (thumbnail) kích thước 400x400 định dạng JPEG ngay khi người dùng tải ảnh lên.

### 🛠 Bảo trì & Cải tiến hệ thống
- <kbd>6aa2c03</kbd> **Tái cấu trúc Monorepo**: Sắp xếp lại mã nguồn theo cấu trúc thư mục frontend và backend riêng biệt nhằm quản lý dự án hiệu quả hơn.
- <kbd>0eca387</kbd> **Cấu hình CORS toàn cục**: Thiết lập cấu hình CORS trên backend để cho phép truy cập tài nguyên an toàn giữa các nguồn gốc khác nhau.
- <kbd>1f166d6</kbd> **Tích hợp UI/UX AI**: Thêm thư viện `ui-ux-pro-max-skill` nhằm hỗ trợ các tính năng thông minh trong thiết kế giao diện.
- <kbd>15e10e3</kbd> **Pipeline CI/CD**: Thiết lập quy trình tự động hóa triển khai (CI/CD) và cấu hình Dockerfile để tối ưu hóa quy trình release.
- <kbd>f9367ee</kbd> & <kbd>2bdd2e8</kbd> **Định dạng mã nguồn**: Sử dụng Spotless để tự động chuẩn hóa định dạng mã nguồn theo quy chuẩn chung của dự án.

### 🐛 Sửa lỗi
- <kbd>1f219d3</kbd> **Cấu hình HTTPS**: Điều chỉnh URL server cho Swagger và thiết lập cơ chế tin cậy reverse proxy headers để đảm bảo bảo mật kết nối.
- <kbd>daca16a</kbd> **Sửa lỗi Swagger File Picker**: Khôi phục chức năng chọn tệp tin trên tài liệu Swagger bằng cách sử dụng `@Parameter` với định dạng binary thay vì `@RequestBody`.
