## 🎯 Tổng quan mục tiêu
Bản phát hành này tập trung vào việc hoàn thiện hệ thống quản lý dữ liệu đa phương tiện với việc bổ sung tính năng tải ảnh, trích xuất metadata và mở rộng các API quản lý shortlink. Đồng thời, chúng tôi đã tối ưu hóa kiến trúc backend thông qua việc chuẩn hóa xử lý ngoại lệ và tái cấu trúc các dịch vụ lưu trữ để đảm bảo hệ thống vận hành ổn định và dễ bảo trì hơn.

## ✨ Chi tiết thay đổi

### 🚀 Tính năng mới
- <kbd>a29680c</kbd> **Mở rộng API quản lý Shortlink**: Bổ sung các endpoint hỗ trợ liệt kê (slice), lấy chi tiết, xóa và xem trước thông tin cho các shortlink.
- <kbd>a595400</kbd> **Tích hợp MediaMetadataExtractor**: Tự động trích xuất và lưu trữ thông tin EXIF vào entity Media ngay khi người dùng tải tệp lên.
- <kbd>e16e648</kbd> **Hỗ trợ tải ảnh lên hệ thống**: Triển khai chức năng upload ảnh kèm theo cơ chế lưu trữ tệp tin và duy trì dữ liệu trong entity Media.

### 🐛 Sửa lỗi
- <kbd>45a206c</kbd> **Khắc phục Schema Swagger**: Điều chỉnh cấu hình multipart upload sử dụng `@RequestPart` và chú thích OpenAPI để hiển thị đúng tài liệu API.

### 🛠 Bảo trì hệ thống
- <kbd>eb038d4</kbd> **Cập nhật phương thức truy xuất preview**: Chuyển đổi dữ liệu xem trước sang định dạng `MediaPreview` để tăng tính đồng bộ và chặt chẽ trong code.
- <kbd>d829601</kbd> **Tối ưu logic I/O tệp tin**: Di chuyển logic xử lý I/O của file preview vào tầng service thông qua bản ghi `MediaPreview`.
- <kbd>b7c35a2</kbd> **Tập trung xử lý ngoại lệ**: Chuyển logic xử lý lỗi vào `GlobalExceptionHandler` nhằm làm gọn mã nguồn tại các controller.
- <kbd>7bd0489</kbd> **Thay đổi kiểu dữ liệu ID**: Chuyển đổi định dạng ID từ UUID sang Long trong entity Album để tối ưu hóa hiệu suất truy vấn.
- <kbd>75967a0</kbd> **Tự động hóa timestamp**: Cập nhật các lớp entity sử dụng `@CreationTimestamp` để hệ thống tự động ghi nhận thời gian tạo bản ghi.
- <kbd>846bdc0</kbd>, <kbd>a9aacd5</kbd> **Định dạng mã nguồn**: Tự động định dạng code bằng Spotless để đảm bảo tuân thủ quy chuẩn coding convention của dự án.
