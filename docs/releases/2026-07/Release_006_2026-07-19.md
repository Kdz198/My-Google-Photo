## 🎯 Tổng quan mục tiêu
Bản phát hành này tập trung xây dựng nền tảng dữ liệu cốt lõi cho hệ thống quản lý Media và Album thông qua việc thiết lập các thực thể (entities) và cấu hình cơ sở dữ liệu. Đồng thời, quy trình phát triển được tối ưu hóa thông qua việc tự động hóa kiểm soát định dạng mã nguồn, tài liệu API và ghi chú phát hành để tăng hiệu suất đội ngũ.

## ✨ Chi tiết thay đổi

### 🚀 Tính năng mới
- <kbd>95c2947</kbd> **Cấu hình định danh duy nhất**: Tích hợp UUID vào thực thể AlbumMedia để đảm bảo tính định danh nhất quán cho các mục dữ liệu.
- <kbd>fc2aac4</kbd> **Khởi tạo quản lý Album**: Xây dựng cấu trúc thực thể Album, AlbumMedia và repository tương ứng để hỗ trợ chức năng quản lý album ảnh.
- <kbd>c867134</kbd> **Khởi tạo thực thể Media**: Thiết lập cấu trúc dữ liệu cơ bản cho Media và repository phục vụ nhu cầu lưu trữ nội dung.
- <kbd>0cb5732</kbd> **Cấu hình kết nối cơ sở dữ liệu**: Cập nhật các thông số thiết lập trong `application.properties` để kết nối hệ thống với database.
- <kbd>a06395d</kbd> **Tích hợp tài liệu API**: Thêm thư viện springdoc-openapi nhằm tự động hóa việc tạo tài liệu API cho các dịch vụ.
- <kbd>3687d2d</kbd> **Thiết lập công cụ tự động hóa**: Cấu hình Spotless để định dạng mã nguồn tự động và tích hợp cơ chế tự động tạo ghi chú phát hành.

### 🛠 Bảo trì hệ thống
- <kbd>e3ff207</kbd> **Hoàn thiện thực thể Media**: Tối ưu hóa thực thể Media sau khi áp dụng Lombok và cập nhật các kiểu dữ liệu JdbcTypeCode.
- <kbd>fc731e1</kbd> **Refactor thực thể Media**: Tái cấu trúc Media entity bằng Lombok, chuyển đổi định dạng ID sang Long và bổ sung MediaMetadata với các chú thích kỹ thuật.
- <kbd>577ee75</kbd> **Cập nhật quy trình CI**: Điều chỉnh các workflows CI để đảm bảo quy trình định dạng mã nguồn và tạo ghi chú phát hành hoạt động ổn định.
