## 🎯 Tổng quan mục tiêu
Bản phát hành này tập trung vào việc nâng cao trải nghiệm quản lý dữ liệu đa phương tiện thông qua việc bổ sung các tính năng xử lý ảnh tự động và hoàn thiện các endpoint quản lý shortlink. Đồng thời, hệ thống cũng được tối ưu hóa kiến trúc xử lý lỗi, cải thiện tài liệu API và thiết lập nền tảng CI/CD nhằm đảm bảo tính ổn định và khả năng mở rộng trong tương lai.

## ✨ Chi tiết thay đổi

### 🚀 Tính năng mới
- <kbd>7d8b207</kbd> **Tự động tạo Thumbnail**: Bổ sung `ThumbnailService` để tự động tạo ảnh thu nhỏ kích thước 400x400 định dạng JPEG khi tải lên.
- <kbd>a29680c</kbd> **Quản lý Shortlink**: Triển khai các endpoint hỗ trợ liệt kê (dạng slice), xem chi tiết, xóa và xem trước shortlink.
- <kbd>a595400</kbd> **Trích xuất Metadata**: Tích hợp `MediaMetadataExtractor` để tự động đọc thông tin EXIF và gắn vào thực thể Media khi người dùng tải lên.

### 🐛 Sửa lỗi
- <kbd>1f219d3</kbd> **Cấu hình HTTPS**: Cấu hình URL máy chủ HTTPS cho Swagger và thiết lập cơ chế tin cậy cho các header từ reverse proxy.
- <kbd>daca16a</kbd> **Sửa lỗi chọn file Swagger**: Thay đổi cách truyền dữ liệu bằng cách sử dụng `@Parameter` với định dạng binary thay vì `@RequestBody` để khôi phục trình chọn file.
- <kbd>45a206c</kbd> **Cấu hình Upload Multipart**: Sửa lỗi schema upload multipart trong Swagger bằng cách kết hợp `@RequestPart` và các chú thích OpenAPI.

### 🛠 Bảo trì hệ thống
- <kbd>15e10e3</kbd> **Thiết lập CI/CD**: Thêm quy trình (pipeline) CI/CD và Dockerfile để chuẩn hóa quy trình triển khai ứng dụng.
- <kbd>eb038d4</kbd> **Tối ưu hóa Preview**: Cập nhật logic truy xuất dữ liệu xem trước (preview) để sử dụng kiểu dữ liệu `MediaPreview`.
- <kbd>d829601</kbd> **Tái cấu trúc I/O**: Chuyển logic xuất nhập file (I/O) của tính năng xem trước sang lớp Service thông qua `MediaPreview` record.
- <kbd>b7c35a2</kbd> **Đơn giản hóa Controller**: Di chuyển logic xử lý ngoại lệ về `GlobalExceptionHandler` giúp code Controller trở nên tinh gọn và dễ bảo trì hơn.
