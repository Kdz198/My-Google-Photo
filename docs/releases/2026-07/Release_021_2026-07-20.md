## 🎯 Tổng quan mục tiêu
Bản phát hành này tập trung vào việc mở rộng hệ sinh thái ứng dụng với sự hỗ trợ của Flutter, đồng thời tối ưu hóa quy trình xử lý media và cải thiện trải nghiệm API thông qua tài liệu Swagger chính xác hơn. Hệ thống cũng được nâng cấp về kiến trúc và CI/CD, giúp tăng tính ổn định, khả năng bảo trì và tốc độ phát triển cho các tính năng trong tương lai.

## ✨ Chi tiết thay đổi

### 🚀 Tính năng mới
- <kbd>[80baa46]</kbd> **Tích hợp Flutter**: Khởi tạo cấu trúc dự án theo mô hình mono-repo, hỗ trợ tích hợp ứng dụng giao diện người dùng bằng Flutter.
- <kbd>[7d8b207]</kbd> **Dịch vụ Thumbnail**: Bổ sung `ThumbnailService` giúp tự động tạo ảnh thu nhỏ (thumbnails) định dạng JPEG với kích thước 400x400 khi người dùng tải tệp lên.
- <kbd>[a29680c]</kbd> **Mở rộng API Shortlink**: Cập nhật các endpoint mới hỗ trợ quản lý liên kết rút gọn bao gồm: xem danh sách (dạng slice), xem chi tiết, xóa và xem trước liên kết.

### 🛠 Bảo trì hệ thống & Cải tiến
- <kbd>[15e10e3]</kbd> **Thiết lập CI/CD**: Cấu hình pipeline CI/CD tự động hóa quy trình triển khai và bổ sung Dockerfile để tối ưu hóa việc đóng gói ứng dụng.
- <kbd>[eb038d4]</kbd> **Tối ưu hóa Preview**: Chuyển đổi logic truy xuất bản xem trước (preview) sang sử dụng kiểu dữ liệu `MediaPreview` giúp đồng bộ hóa dữ liệu tốt hơn.
- <kbd>[d829601]</kbd> **Refactor xử lý tệp**: Tái cấu trúc logic xuất/nhập tệp xem trước, chuyển toàn bộ vào lớp Service thông qua cấu trúc `MediaPreview` record.
- <kbd>[b7c35a2]</kbd> **Chuẩn hóa xử lý lỗi**: Chuyển logic xử lý ngoại lệ vào `GlobalExceptionHandler` giúp mã nguồn Controller tinh gọn và chuyên nghiệp hơn.

### 🐛 Sửa lỗi
- <kbd>[1f219d3]</kbd> **Cấu hình HTTPS**: Thiết lập lại URL server HTTPS cho Swagger và cấu hình tin cậy cho các tiêu đề (headers) của reverse proxy.
- <kbd>[daca16a]</kbd> **Sửa lỗi chọn tệp Swagger**: Khôi phục chức năng chọn tệp trong Swagger bằng cách sử dụng `@Parameter` với định dạng binary thay vì `@RequestBody`.
- <kbd>[45a206c]</kbd> **Cấu hình Multipart Upload**: Cập nhật schema tải tệp lên đa phần (multipart) trong Swagger thông qua `@RequestPart` và các chú thích OpenAPI để hiển thị đúng tài liệu API.
